Dive into Deep Learning | 1. Introduction

Dec 31, 2023 in Deep Learning

Introduction

머신러닝은 규칙이나 인간의 창의력만으로 해결하기 어려운 작업(ex. 날씨 예측, 텍스트 질문 대답, 이미지 식별, 추천 프로그램 등)에 적합하다. 알려지지 않은 원리를 계산하기 위해 경험을 통해 학습할 수 있는 알고리즘에 중점을 두고 있으며, 경험을 쌓을수록 성능이 향상된다.

Programming with Data란 원하는 작업을 직접 코딩하는 대신 레이블이 지정된 데이터셋을 제공하여 머신이 학습할 수 있는 프로그램을 작성하는 것을 말한다.

Key Components

  • 머신 러닝의 핵심 구성 요소
    1. Data
      • input이 fixed-length vectors일 때 vector의 길이 = the dimensionality of the data
      • 모든 데이터가 fixed-length vectors로 표현될 수 있는 것은 아님
      • 딥러닝은 varying-length data를 처리하는 데 뛰어난 성능을 보임
      • 데이터가 많을수록 모델 훈련이 더 쉬워지고 사전 가정에 덜 의존할 수 있음
      • 데이터가 많아도 오류가 있거나 선택한 feature가 target 예측에 유용하지 않으면 학습이 실패할 수 있음
    2. Models
      • 딥러닝은 주로 연속적인 데이터 변환을 적용하는 강력한 모델 집합에 초점을 맞춘다는 점에서 기존 접근 방식과 차별화됨
    3. Objective Functions
      • 모델 성능을 측정하기 위한 measure 필요, 이를 objective function 또는 loss function이라 정의, 낮을수록 좋음
      • 수치 값 예측에서 가장 흔한 loss function = squared error, classification의 objective = error rate 최소화
      • errror rate 등은 미분이 불가능하거나 기타 복잡성으로 직접 최적화가 어렵기 때문에 surrogate objective를 최적화하기도 함
      • loss를 모델 parameter의 함수로, 훈련 데이터셋을 상수로 두고 parameter 최적값들을 학습
      • training 데이터에서 잘 수행되더라도 보이지 않은 데이터에 대해 일반화되지 못할 때, 이를 training 데이터에 overfit되었다고 말함
    4. Optimization Algorithms
      • 딥러닝에서 인기 있는 최적화 알고리즘은 gradient descent 기법을 기반으로 함
      • 각 단계에서 각 parameter를 미세하게 변경한 경우 loss가 어떻게 변화하는지 확인, loss를 줄이는 방향으로 parameter 업데이트

Kinds of Machine Learning Problems

Supervised Learning

  • feature-label 쌍 = example로 구성된 데이터셋을 모델에 제공하여 모델 parameter를 선택
  • 입력과 출력의 종류, 크기 및 양에 따라 모델링 결정이 크게 달라짐
  1. Regression
    • 가장 간단한 지도 학습 작업 중 하나로, 예측하려는 대상이 임의의 수치값을 가질 때 사용
    • 이 책에서는 대부분 squared error loss function을 최소화하는 것에 중점을 둠(데이터가 가우시안 노이즈에 의해 손상되었다고 가정)
  2. Classification
    • example이 어떤 category(또는 class)에 속하는지 예측하는 작업, 2개의 클래스만 있는 경우 binery classification
    • classification 문제의 가장 일반적인 loss function은 cross-entropy
    • 모델을 확률로 표현하여 각 class에 대한 확률을 출력하도록 하면 더 많은 불확실성 정보를 얻을 수 있음
  3. Tagging
    • 각 example이 상호 배타적이지 않은 class 예측하는 학습 문제를 multi-label classification라고 함
    • 이미지에 여러 label이 할당될 수 있는 상황을 다루는데 유용, auto-tagging 문제가 대표적
  4. Search
    • information retrieval 분야에서 쿼리와 관련된 결과 집합 중 어떤 것을 특정 사용자에게 가장 눈에 띄게 표시해야 하는지를 결정하는 것이 목표
    • 현재 검색 엔진은 머신러닝 및 행동 모델을 사용하여 쿼리에 따른 관련성 점수를 얻음
  5. Recommender Systems
    • 검색 및 순위 지정과 관련된 문제 설정으로 목표는 비슷하나 사용자 맞춤화를 강조한다는 것이 차이점
    • 명시적 피드백 또는 암시적 피드백 제공, 예상 별점이나 특정 사용자가 특정 항목을 구매할 확률과 같은 점수를 추정
    • 검열, 인센티브, 피드백 루프를 어떻게 처리할 것인지에 관한 이러한 문제들은 여전히 연구 진행 중
  6. Sequence Learning
    • 입력의 시퀀스를 수집하거나 출력의 시퀀스(또는 둘 다)를 방출하는 모델
    • sequence-to-sequence 학습은 입력과 출력이 모두 variable-length sequences로 구성된 문제
    • 태그 지정 및 구문 분석(텍스트 시퀀스에 주석을 다는 작업, 입출력이 같은 순서로 정렬되어 있음), 음성-텍스트 변환(입력 > 출력), 텍스트-음성 변환(입력 < 출력), 기계 번역(입출력 시퀀스 길이가 다르며 정렬되어 있지 않음),

Unsupervised and Self-Supervised Learning

  1. clustering - 데이터 그룹화
  2. subspace estimation - 데이터 속성을 포착하는 소수의 파라미터 찾기, 의존성이 선형인 경우 principal component analysis
  3. causality, probabilistic graphical model - 데이터의 원인, 연관 관계 설명
  4. deep generative model - variational autoencoder 발명으로 시작, generative adversarial network 개발로 이어짐
  5. self-supervised learning - 텍스트 빈칸 채우기, 이미지 가려진 부분 예측, 주로 표현을 학습하며 이후 파인튜닝하여 사용

Interacting with an Environment

  • supervised & unsupervised learning은 알고리즘이 환경과 연결이 끊긴 후에 이루어지기 때문에 offline learning이라고도 함
  • offline learning은 동적인 환경의 복잡성을 걱정할 필요 없이 패턴 인식만을 분리하여 고민할 수 있다는 장점이 있으나, 예측뿐만 아니라 행동을 선택하는 지능형 에이전트를 훈련하려면 에이전트의 행동이 향후 관찰에 어떤 영향을 미칠지 고려해야 함
  • 환경과의 상호작용을 고려하면 training 데이터와 test 데이터의 분포가 서로 다른 distribution shift 문제 발생
  • 에이전트가 환경과 상호 작용하는 학습 문제를 다루기 위한 프레임워크가 reinforcement learning

Reinforcement Learning

  • 로봇 공학, 대화 시스템, 심지어 비디오 게임용 인공 지능(AI) 개발에 적용
  • 강화 학습 문제에 딥러닝을 적용한 것이 Deep reinforcement learning
  • 프로세스 - 환경으로부터 관찰 정보 수신 ⇒ 동작 선택 ⇒ 매커니즘(actuator)을 통해 환경으로 다시 전송 ⇒ 각 루프가 끝나면 환경으로부터 보상을 받음
  • 에이전트의 행동은 policy에 의해 제어됨, policy는 환경 관찰을 행동으로 매핑하는 함수, Reinforcement Learning의 목표는 좋은 policy을 생성하는 것
  • 결과에 대해 어떤 행동을 칭찬하거나 비난할지 결정하는 credit assignment 문제, 부분적 관찰 가능성 문제를 해결해야 함
  • 현재 가장 좋은 전략을 policy 활용할지, 단기적인 보상을 포기하고 전략의 영역을 탐색할지 끊임없이 선택해야 함
  • 강화 학습의 문제
    • 환경이 완전히 관찰되는 경우 - Markov decision process
    • 상태가 이전 행동에 의존하지 않는 경우 - contextual bandit problem
    • 상태가 없고 처음 보상이 알려지지 않은 가능한 행동 집합만 있는 경우 - multi-armed bandit problem

Roots

  • 딥러닝은 최근에 등장했지만 데이터로부터 학습하는 핵심 아이디어는 수세기 동안 연구되어 왔으며, 데이터를 분석하고 미래의 결과를 예측하고자 하는 인간의 욕구는 많은 자연과학 및 수학의 근간을 이루고 있음
  • 통계는 데이터의 가용성과 수집을 통해 비약적으로 발전, 그 외 정보/계산 이론, 신경과학/심리학이 머신러닝 발전에 영향

The Road to Deep Learning

  • 랜덤 액세스 메모리는 데이터 증가 속도를 따라잡지 못함 & 계산 능력 향상은 데이터셋 증가를 앞지름 ⇒ 통계 모델이 메모리 효율성을 높여야 한다는 것을 의미
  • GPU를 통해 컴퓨팅 예산이 증가함에 따라 parameter를 최적화하는 데 더 많은 컴퓨터 주기를 사용할 수 있게 됨 ⇒ deep neural network의 부상

Success Stories

  • 인간 제작자를 의도적으로 조종할 수 있는 지각 있는 AI 시스템 개발은 먼 이야기
    • 첫째, AI 시스템은 특정 목표 지향적인 방식으로 설계, 훈련 및 배포되기 때문에 행동은 일반적인 지능처럼 보일 수 있지만, 그 설계의 기반은 규칙, 휴리스틱 및 통계 모델의 조합
    • 둘째, 현재로서는 스스로 개선하고 스스로 추론할 수 있으며 일반적인 작업을 해결하면서 아키텍처를 수정, 확장, 개선할 수 있는 _artificial general intelligence_을 위한 도구가 존재하지 않음
  • 통계 모델을 주의 없이 적용할 경우 인종, 성별, 연령 편견을 초래할 수 있으며, 결과적 의사결정을 자동화할 경우 절차적 공정성에 대한 합리적인 우려가 제기될 수 있음, 이러한 알고리즘을 신중하게 사용하는 것이 중요

The Essence of Deep Learning

  • 딥러닝의 차별점은 데이터로부터 표현된 여러 계층 각각에서 이뤄지는 연산이 공동으로 학습된다는 점
  • 딥러닝이 도입되면서 feature extractor가 자동으로 튜닝된 필터로 대체되어 정확도가 크게 향상
  • 딥러닝의 장점
    • 기존 학습 파이프라인의 끝에 있는 얕은 모델뿐만 아니라 노동 집약적인 피처 엔지니어링 프로세스도 대체할 수 있음
    • 도메인별 전처리의 상당 부분을 대체함으로써 이전에 컴퓨터 비전, 음성 인식, 자연어 처리, 의료 정보학 및 기타 애플리케이션 영역을 구분하던 경계를 없애고 다양한 문제를 해결할 수 있는 통합된 툴셋 제공
